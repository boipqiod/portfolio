Web

중계 서버의 인증 대기 페이지가 기존에는 Polling 방식으로 구현되어 있어, SSE 방식으로 리팩토링했습니다.
SSE 방식으로 리팩토링하면서 서버 부하를 줄이고, 인증 대기 페이지의 응답 속도를 높였습니다.
또한, 리팩토링을 진행하며 인증 성공률 또한 높일 수 있었습니다.

백오피스에 결제 시스템을 도입하며 결제 시스템과 연동하는 작업을 진행했습니다.
프론트엔드 작업뿐 아니라, API 서버 내 결제 시스템과 연동 모듈 작업 또한 진행했습니다.
처음 겪어본 JSON 구조를 사용하지 않고, 쿼리스트링을 이용한 API였기 때문에 처음에는 고전을 면치 못했습니다.
이를 극복해나가는 과정이 힘들었지만, 알지 못하는 부분이 많다고 느꼈고, 더 공부해야겠다 느꼈습니다.

마침내 개발을 완료한 뒤 불가능하다고 생각했던 기능을 구현해 냈을 때의 성취감은 말로 표현할 수 없었습니다.

회고

기존에는 React와 Express를 이용해 서버에서 Youtube API를 이용한 채팅 읽기, 채팅 전송 기능을 구현했습니다.
하지만, Youtube API는 각 사용자 마다 API Key를 받아야되고, 수량도 한정적인 한계로 인해 Electron으로 리팩토링을 진행했습니다.
Electron은 처음 사용해보는 기술이었기 때문에, Electron 메인 프로세스를 이해하지 못하고 많이 헤맸습니다.
점차 익숙해져, 지금은 Electron과 React만으로 개발을 진행하고 있습니다.

React를 제대로 사용해본 첫 프로젝트입니다.
원래도 코드를 모듈화하고 재사용성을 높이는 것을 좋아했지만, React를 사용하면서 더욱 코드를 모듈화하고 재사용성을 높이는 것에 대해 공부하게 되었습니다.
이런 경험으로 자신만의 코드 스타일이 만들어 지는 것을 느꼈습니다.

크로스 플랫폼을 사용해보니, 프론트엔드 개발자를 선택하길 잘했다고 느꼈습니다.
프론트엔드 개발자는 웹, 앱, 데스크탑 등 다양한 플랫폼에서 개발을 진행할 수 있기 때문에, 더욱 다양한 사용자를 경험할 수 있다고 생각합니다.
